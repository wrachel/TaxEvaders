Reflection

Overall, I thought taht this FRQ was relatively easy. The key thing was mostly for part a, which was using recursion. Part b was fairly simple and did not really test anything new.

Key Takeaways:

Recursion. Also, used a little bit of wrapper classes here in order to convert integer to string easier (not necessary for CB solution but necessary for MVC implementation).

Solution

public class NumberSystem { part a) public static int gcf(int a, int b){ int temp = a % b; if(temp == 0){ return b; } else{ return gcf(b, temp); //recursion } } part b) public static void reduceFraction(int numerator, int denominator){ if(numerator % denominator = 0){ System.out.println(numerator / denominator); } else{ int gcf = NumberSystem.gcf(numerator, denominator); System.out.println(numerator/gcf + "/" + denominator/gcf); } } }
