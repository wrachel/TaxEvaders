Obligatory Reflection

Maybe doing these in second trimester wasn't a good idea, because I completely forgot what this unit was supposed to be about. Most of my solutions were very elegant by normal standards, but I'm pretty sure they were not the intended solutions. At all. All of my friends used like substring concatenation and stuff, while I was using replace statements.

Speaking of, replace statements are entirely turing complete. This was proved a long time ago but I don't remember who did it. I actually made an entire arithmetic implementation using solely string replacement instructions.

Anyway, I'm getting off-topic. The activity was a pretty nice review of...objects, I guess? It was a lot of getter and setter stuff. And object instantiation. Lots of that, actually. I don't really have much to say about this -- it's not like this is a particularly groundbreaking revelation. If you're just learning about objects now, you probably aren't cut out fo computer science.

The website creation process was pretty interesting. I had to do a lot of modification to the original code in order to get it to work. That was a headache and a half. I probably could have handled the actual data transfer better, like with an array or a JSON object. I mean, not that creating 6 different parameters isn't fun, but it's a pain to read and a lot of copying and pasting. It also violates DRY, which is Very Bad (tm).
